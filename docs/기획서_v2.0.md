## 1. 프로젝트 개요

### 1.1 한 줄 요약

**Gemini처럼 대화하면서 광고 업무를 처리하는 데스크톱 AI 어시스턴트. 비용 0원.**

### 1.2 왜 만드는가

형(광고 마케터)이 현재 Gemini 웹사이트를 통해 업무를 처리하고 있지만:

- 매번 브랜드/타겟/톤앤매너를 처음부터 설명해야 함
- 이전 작업을 찾아서 이어가기 어려움
- 결과물을 일일이 복사해서 Word에 붙여넣기
- 매체별 규격을 직접 기억해야 함
- Gemini 유료 전환 시 비용 부담

4Bro는 **Gemini의 두뇌 + 광고 업무에 특화된 전용 인터페이스**를 결합하여, 형이 지금보다 훨씬 편하게 작업할 수 있게 만든다.

### 1.3 핵심 목표

- **대화형 AI 어시스턴트** — 채팅으로 자연스럽게 업무 처리
- **자율 에이전트 기능** — 복잡한 작업은 AI가 알아서 다단계 실행
- **프로젝트 기억** — 브랜드/타겟/톤을 기억하고 매번 다시 설명할 필요 없음
- **비용 0원** — 무료 API + 로컬 모델 백업으로 지속 사용 가능
- **비전문가 사용 가능** — exe 실행으로 바로 사용

### 1.4 v1.1 대비 변경 요약

| 항목 | v1.1 | v2.0 |
|------|------|------|
| 사용 방식 | 탭별 폼 입력 → 버튼 클릭 | **자연어 대화 (채팅)** |
| AI 엔진 | 로컬 Ollama만 사용 | **Gemini API 메인 + 로컬 백업** |
| 이미지 | 불가 | **분석 + 생성 가능** |
| 자율 실행 | 없음 | **에이전트 모드 (다단계 자동)** |
| 프로젝트 관리 | 없음 | **브랜드 기억, 대화 히스토리** |
| 웹 검색 | 불가 (오프라인) | **경쟁사 조사, 트렌드 검색** |
| 네트워크 | 완전 오프라인 | **온라인 (무료 API 활용)** |

## 2. 실제 사용자 업무 분석

형이 Gemini에서 실제로 했던 작업들을 분석하여 기능을 설계했다.

### 2.1 업무 패턴 A: 광고 카피 핑퐁

```
형: "묵혼온라인 신서버 오픈인데, 사전예약 끝나고 추가등록 기간이야.
     버튼 문구를 뭐라고 하면 좋을까? (상황 상세 설명)"
AI:  여러 방향 제안 (혜택 강조형, 신서버 강조형, 추가등록 활용형...)
형: "강호 합류하기 vs 지금 즉시 합류 뭐가 나아?"
AI:  비교 분석 + 추천
형: "강호합류 등록하기는? 말이 이상하지 않나?"
AI:  어색한 이유 분석 + 대안 제시
형: "강호 합류하기로 가자. GFA용, 인벤용으로도 변형해줘"
AI:  매체별 규격에 맞춰 변형 제공
```

**핵심 요구**: 대화 맥락 유지, A/B 비교, 매체별 변형, 톤 조절

### 2.2 업무 패턴 B: 영상 광고 기획

```
형: (배너 이미지 4장 첨부) "1,4번은 레퍼런스고 2,3번이 내 배너야.
     이걸 영상 광고로 만들고 싶어. 불나오고 반짝거리는 효과도 넣고 싶거든"
AI:  영상 구성 제안 (레이아웃, 효과, 트랜지션)
형: "위에 얘기한 구성으로 이미지 3개 만들어서 보여줘"
AI:  레퍼런스 이미지 생성 → 디자이너 전달용 가이드 작성
```

**핵심 요구**: 이미지 첨부 분석, 이미지 생성, 디자인 가이드 작성

### 2.3 업무 패턴 C: 대량 작업 (에이전트가 필요한 순간)

형이 지금까지 수동으로 했지만, 자동화하면 좋을 작업:

- "이 카피를 GFA, GDN, 유튜브, 인벤용으로 **전부** 만들어줘" → 매체별 일괄 변형
- "신서버 오픈 캠페인 **전체 패키지** 만들어줘" → 타겟분석+카피+SNS+기획서
- "경쟁사 무협게임 마케팅 **조사해줘**" → 웹 검색 + 정리 + 분석
- "2월 캠페인 데이터로 **보고서** 만들어줘" → 데이터 분석 + 보고서 자동 생성

## 3. 기능 명세

### 3.1 채팅 모드 (기본)

Gemini/ChatGPT처럼 자연어로 대화하면서 작업하는 기본 모드.

| 기능 | 설명 |
|------|------|
| 자연어 대화 | 상황 설명 → AI 제안 → 수정 요청 → 확정 (핑퐁) |
| 대화 맥락 유지 | 한 대화 안에서 이전 내용 기억 |
| 이미지 첨부 분석 | 배너, 광고 소재 등 이미지를 붙여서 분석 요청 |
| 이미지 생성 | "이런 느낌의 레퍼런스 만들어줘" → 이미지 생성 |
| 파일 첨부 분석 | PDF, Word, txt 파일 첨부 → 내용 분석 |
| 실시간 스트리밍 | AI 응답을 한 글자씩 실시간 표시 |

#### 사용 예시

```
형: 묵혼온라인 신서버 곤륜 오픈 광고 카피 만들어줘.
    타겟은 4050 남성이고 무협 세계관이야.

4Bro: ## 헤드라인 제안
      1. 강호의 전설이 다시 시작된다 - 곤륜 서버 대개방
      2. 마침내, 당신이 꿈꾸던 무림이 열렸다
      ...

      특정 매체에 맞게 변형이 필요하시면 말씀해 주세요.

형: 2번이 좋은데 좀 더 비장하게 바꿔줘

4Bro: "마침내, 협객의 귀환 - 곤륜이 당신을 부른다"
      이런 느낌은 어떠세요?
```

### 3.2 에이전트 모드 (선택)

복잡한 작업을 AI가 자율적으로 다단계 실행하는 모드. 각 단계를 자동으로 진행하되, 진행 상황을 실시간으로 보여준다.

| 워크플로우 | 단계 | 설명 |
|-----------|------|------|
| **매체별 일괄 변형** | 1. 원본 카피 확인 → 2. 매체별 규격 적용 → 3. 톤 조절 → 4. 전체 출력 | 하나의 카피를 여러 매체용으로 한번에 변형 |
| **캠페인 패키지** | 1. 타겟 분석 → 2. 전략 수립 → 3. 광고 카피 → 4. SNS 콘텐츠 → 5. 기획서 | 캠페인 전체 산출물 자동 생성 |
| **문서분석→전략** | 1. 문서 분석 → 2. 인사이트 도출 → 3. 전략 제안 | 첨부 문서 기반 전략 자동 도출 |
| **경쟁사 리서치** | 1. 웹 검색 → 2. 정보 수집 → 3. 비교 분석 → 4. 차별화 전략 | 경쟁사 마케팅 자동 조사 |
| **보고서 자동화** | 1. 데이터 분석 → 2. 성과 요약 → 3. 인사이트 → 4. 제안 | 캠페인 결과 보고서 자동 작성 |
| **카피 대량 생성** | 1. 방향 분석 → 2. 대량 생성 → 3. 분류/정리 | 헤드라인 50개, 바디카피 20개 등 |

#### 사용 예시

```
형: 묵혼 곤륜 서버 캠페인 패키지 만들어줘

4Bro: 캠페인 패키지 생성을 시작합니다.
      프로젝트 '묵혼온라인'의 저장된 정보를 사용합니다.

      ✅ Step 1/5: 타겟 분석
         4050 남성, MMORPG 코어 유저, 무협 세계관 선호...

      ⏳ Step 2/5: 전략 수립
         정통 무협 감성 소구 + 현물 보상 강조 전략...

      ○ Step 3/5: 광고 카피 (대기)
      ○ Step 4/5: SNS 콘텐츠 (대기)
      ○ Step 5/5: 캠페인 기획서 (대기)

(각 단계가 자동으로 진행되며, 완료 후 Word 자동 저장)
```

#### 에이전트 모드 제어

- **자동 실행**: 시작하면 끝까지 자동 진행
- **단계별 확인**: 각 단계 완료 후 "계속할까요?" 확인 (옵션)
- **중간 중단**: 언제든 중단 가능
- **결과 수정**: 완료 후 특정 단계만 재생성 가능

### 3.3 프로젝트 관리

#### 3.3.1 브랜드/프로젝트 프로필

프로젝트별로 핵심 정보를 저장해두면, 매번 설명하지 않아도 AI가 알고 있다.

```
[프로젝트: 묵혼온라인]
- 장르: 무협 MMORPG
- 타겟: 4050 남성
- 톤앤매너: 정통 무협, 비장함, 강호
- 주요 KPI: 사전예약/등록 수
- 경쟁사: 천애명월도, 검은사막
- 주요 USP: 순금 경품, 유일무기, 쾌속 성장
```

→ 대화할 때 "묵혼 프로젝트에서" 라고만 하면 위 정보가 자동 주입됨

#### 3.3.2 대화 히스토리

- 프로젝트별로 대화 목록 관리
- 이전 대화 검색/재개 가능
- "지난번 곤륜 카피 작업 이어서 하자" 가능

#### 3.3.3 카피 북마크/확정

- 대화 중 마음에 드는 카피를 "확정" 표시
- 프로젝트별 확정된 카피 목록 관리
- 나중에 확정 카피만 모아서 Word로 내보내기

### 3.4 스마트 기능

#### 3.4.1 매체별 규격 자동 적용

AI가 매체별 광고 규격을 알고 있어서, 매체만 지정하면 자동으로 맞춤 변형한다.

| 매체 | 헤드라인 | 설명문 | 특징 |
|------|---------|--------|------|
| 네이버 GFA | 25자 | 45자 | 짧고 임팩트 |
| Google GDN | 30자 | 90자 | 정보 전달형 |
| 유튜브 | 제한 없음 | 제한 없음 | 영상 연계 |
| 인벤 배너 | 20자 | 30자 | 게임 커뮤니티 톤 |
| 인스타그램 | - | 2,200자 | 감성 + 해시태그 30개 |
| 페이스북 | - | 제한 없음 | 정보 전달형 긴 글 |

#### 3.4.2 광고 전문가 / 범용 모드

| 모드 | 설명 | AI 성격 |
|------|------|--------|
| 광고 전문가 | 광고/마케팅 업무 특화 | 광고 전략가, 카피라이터 역할 |
| 범용 어시스턴트 | 일반 업무 보조 | 이메일, 보고서, 번역, 질의응답 |

#### 3.4.3 웹 검색

온라인 환경을 활용하여:
- 경쟁사 마케팅 동향 조사
- 업계 트렌드/키워드 수집
- 레퍼런스 검색

#### 3.4.4 결과물 관리

- **자동 저장**: 에이전트 모드 실행 결과는 자동으로 Word 저장
- **수동 저장**: 대화 중 언제든 Word/txt로 저장
- **최근 파일**: 사이드바에서 최근 저장 파일 바로 열기
- **저장 경로**: `내 문서/4Bro/프로젝트명/` 폴더에 자동 정리

## 4. 시스템 아키텍처

### 4.1 엔진 구조: 3단계 자동 전환

비용 0원을 유지하면서 안정성을 보장하기 위해 3단계 백업 체인을 사용한다.

```
사용자 메시지
    ↓
[1순위] Gemini API (무료 티어)
  - 텍스트 생성, 이미지 분석, 이미지 생성, 웹 검색
  - 빠르고 고품질, 100만 토큰 문맥
  - 한도: 분당 10회, 하루 250회 (Flash 기준)
    ↓ (한도 초과 시)
[2순위] Groq API (무료 티어)
  - 텍스트 생성만 (이미지 불가)
  - 매우 빠른 응답 속도
    ↓ (한도 초과 시)
[3순위] 로컬 Ollama (Qwen 14B)
  - 텍스트 생성만 (이미지 불가)
  - 느리지만 무제한, 오프라인 가능
    ↓
응답 → 사용자에게 표시
```

전환 시 사용자에게 알림: "API 한도 초과로 로컬 모델로 전환되었습니다"

### 4.2 각 엔진별 역할

| 기능 | Gemini API | Groq API | 로컬 Ollama |
|------|-----------|----------|------------|
| 텍스트 대화 | O | O | O |
| 이미지 분석 | O | X | X |
| 이미지 생성 | O | X | X |
| 웹 검색 | O (또는 별도) | X | X |
| 긴 문서 처리 | O (100만 토큰) | O | 제한적 (4K~8K) |
| 에이전트 모드 | O (추론 안정) | 제한적 | 제한적 |
| 속도 | 빠름 | 매우 빠름 | 느림 (2~15 tok/s) |

### 4.3 API 무료 한도 (2026년 3월 기준)

| API | 모델 | 분당 | 하루 | 비고 |
|-----|------|------|------|------|
| Gemini | 2.5 Flash | 10회 | 250회 | 텍스트+이미지 |
| Gemini | 이미지 생성 | - | 500장 | Imagen |
| Groq | Llama 등 | 30회 | 14,400회 | 텍스트만 |

형의 일반적 사용량 (하루 50~100회 텍스트, 10~15장 이미지)은 충분히 커버된다.

> **주의**: Google이 무료 한도를 줄일 수 있음 (2025.12에 50~80% 감소 이력). 로컬 백업이 반드시 필요한 이유.

### 4.4 소프트웨어 스택

| 레이어 | 기술 | 역할 |
|--------|------|------|
| AI 엔진 (메인) | Google Gemini API | 텍스트+이미지 생성/분석 |
| AI 엔진 (백업1) | Groq API | 텍스트 생성 백업 |
| AI 엔진 (백업2) | Ollama + Qwen 14B | 로컬 텍스트 생성 |
| GUI | PyQt6 | 데스크톱 채팅 인터페이스 |
| 문서 처리 | python-docx, PyPDF2 | Word/PDF 읽기/쓰기 |
| 데이터 저장 | SQLite | 대화 히스토리, 프로젝트 프로필 |
| 웹 검색 | Google Search API 또는 SearXNG | 경쟁사 조사, 트렌드 |
| 패키징 | PyInstaller | .exe 배포 |

### 5.4 사용 흐름 (User Flow)

**최초 실행:**
1. 4Bro.exe 실행
2. Gemini API 키 입력 (1회만, 무료 발급)
3. 선택: Ollama 설치 여부 (로컬 백업용, 선택사항)
4. 바로 채팅 시작

**일반 사용:**
1. 프로그램 실행 → 이전 대화 목록 표시
2. 새 대화 시작 또는 이전 대화 이어가기
3. 자연어로 작업 요청
4. AI 응답 확인 → 수정 요청 → 확정
5. 필요시 Word/txt 저장 또는 자동 저장

**에이전트 모드:**
1. "캠페인 패키지 만들어줘" 같은 대규모 작업 요청
2. 또는 사이드바에서 에이전트 워크플로우 선택
3. 자동 실행 시작 → 진행 상황 실시간 표시
4. 완료 시 결과물 자동 저장 + 알림

### 7.2 우선순위

1. **최우선**: 채팅 + Gemini API → 이것만으로도 형이 쓸 수 있는 프로그램
2. **높음**: 프로젝트 기억 + 히스토리 → Gemini 웹보다 나은 이유
3. **높음**: 이미지 분석/생성 → 형의 핵심 업무
4. **중간**: 에이전트 모드 → 대규모 작업 자동화
5. **중간**: 웹 검색, 매체별 규격
6. **낮음**: 패키징, 세부 설정

## 10. 제약사항 및 주의사항

### 10.1 API 의존성

- Gemini API 무료 한도가 변경될 수 있음 (2025.12 감소 이력)

- 이미지 분석/생성은 API 필수

| 기능 | API 모드 | 로컬 모드 |
|------|---------|----------|

| 에이전트 모드 | O (안정) | 제한적 |

- 프로젝트 프로필, 히스토리는 로컬 SQLite에만 저장
